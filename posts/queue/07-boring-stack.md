# The boring stack I actually ship with

When I start a new side project, I am tempted by the new thing. New framework. New runtime. New cloud platform.

Then I remember: I want to *ship*, not learn. So I reach for the boring stack.

- HTML, CSS, vanilla JS (or TypeScript if I have time)
- SQLite for storage. Postgres if I need many writers.
- Go or Node for the API.
- Nginx and a $5 VPS, or just GitHub Pages if it is static.
- Plain bash for deploy.

I have not regretted these choices once. New tools are exciting on day one and painful on day forty. Boring tools are quiet on day one and quiet on day forty.

You can pick a fancy stack later, when the project earns it. Most projects never do. Most projects end on a weekend. Boring tools let you finish the weekend.

Ship first. Learn the shiny thing on your next one.
